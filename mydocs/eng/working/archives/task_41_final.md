# Task 41 Final Report: Fix Saved File Corruption After Clipboard Table Paste

## Objective

Improve the quality of Table controls generated by the `parse_table_html()` function so that HWP files saved after pasting can be opened correctly in Hancom word processor.

## Results

### Key Discovery: char_count_msb Context Dependency

Previous FIX-1 set MSB=true, but when inserting a table into a document with existing content, the "content after the table disappears" issue occurred.

**Cause**: HWP's char_count MSB flag must vary depending on document structure:
- Empty document: MSB=1 (table is the only content)
- Document with existing content: MSB=0 (paragraphs exist after the table)

**Diagnostic method**: Byte-level comparison between cloned existing table insertion (success) vs parse_table_html generated insertion (failure) revealed PARA_HEADER char_count MSB difference.

### Fixed DIFF Items (8 total + FIX-1 re-fix)

| Item | Severity | Fix Content | Effect |
|------|----------|-------------|--------|
| FIX-1 | **Critical** | Table paragraph char_count_msb = false | Content after table renders correctly |
| DIFF-1 | High | &nbsp; -> empty cell handling (cc=1, no PARA_TEXT) | Record structure normalized |
| DIFF-2 | Medium | Added default CharShapeRef to cell paragraphs | Character format reference valid |
| DIFF-3 | Low-Medium | cell_para_shape_id = 0 (default body) | Paragraph format reference valid |
| DIFF-4 | Medium | create_border_fill_from_css() + surgical insert | Border/background reference accurate |
| DIFF-5 | Low-Medium | raw_table_record_attr = 0x04000006 | Cell split prevention always set |
| DIFF-6 | Low | tag=0x00060000, seg_width calculation | Cell LineSeg metrics correct |
| DIFF-7 | Low | Hash-based non-0 instance_id | CTRL_HEADER instance ID valid |
| DIFF-8 | Low | Based on total_height/total_width | Table container LineSeg correct |

### Additional Discovery: DocInfo Re-serialization Bug

Complete DocInfo re-serialization causes file corruption in complex documents. Workaround: keep DocInfo raw_stream and re-serialize only Sections. For new BorderFill/CharShape additions, use surgical insert to partially modify raw_stream.
