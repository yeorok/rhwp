## 1. Goal

Introduce design tokens (CSS variables) into rhwp-studio's CSS and implement responsive layouts usable on mobile browsers.

## 2. Current State Analysis

| Item | Current | Target |
|------|------|------|
| CSS variables | 0 | ~30 tokens |
| Hardcoded colors | 19 types, 40+ lines | Token references |
| Media queries | 0 | 3 breakpoints |
| Touch targets | 22~28px | 44px (mobile) |
| font-size | 13 scattered levels | 5-level system |

## 3. Implementation Plan (4 Steps)

### Step 1: Color Token Adoption

Declare color tokens in `base.css` `:root` and progressively apply to key files.

- Brand colors: `--color-primary`, `--color-primary-dark`, `--color-accent-bg`, etc.
- Surface colors: `--color-surface`, `--color-bg`, `--color-bg-light`
- Borders: `--color-border`, `--color-border-light`, `--color-border-dark`
- Text: `--color-text`, `--color-text-secondary`, `--color-text-muted`
- Interaction: `--color-hover-bg`, `--color-active-bg`, `--color-focus-border`
- Shadows: `--shadow-dropdown`, `--shadow-dialog`

Scope: Start with most frequently used files among 22 CSS files

### Step 2: Typography + Spacing Tokens

- Font sizes: `--font-size-xs`(10px) ~ `--font-size-xl`(16px) 5 levels
- Font families: `--font-family-ui`, `--font-family-mono`
- Spacing: `--space-1`(2px) ~ `--space-8`(16px)
- Sizes: `--radius-sm/md/lg`, `--control-height-sm/md/lg`

### Step 3: Responsive Breakpoints + Layout

Breakpoint definitions:
- Mobile: `~767px`
- Tablet: `768px~1023px`
- Desktop: `1024px~` (maintain current layout)

Mobile layout changes:
- Menu bar → Hamburger menu
- Icon toolbar → Bottom floating or collapsible
- Style bar → Horizontal scroll or collapsible
- Dialogs → Full-screen sheet
- Status bar → Minimal info display
- Editor area → Full screen, pinch zoom

Touch support:
- Button/icon touch target minimum 44px guarantee
- Swipe gestures (menu open/close)
- Layout adjustment on virtual keyboard appearance

### Step 4: Progressive Migration of Existing Code + Verification

- Replace hardcoded values with tokens when modification opportunities arise
- Add mobile viewport scenarios to E2E tests
- Screenshot comparison across major device sizes

## 4. Principles

- **Lightweight tokens**: 30 tokens or fewer, no over-abstraction
- **Progressive migration**: No batch conversion, apply to new code first
- **No tools**: No Sass/PostCSS needed, CSS variables only
- **No dark mode**: Not needed at this point (token system enables future transition)
- **Desktop first**: Add mobile without breaking existing desktop layout

## 5. Verification Methods

- Desktop: All existing E2E test 52 assertions pass
- Mobile: Chrome DevTools device mode + CDP E2E viewport change tests
- Visual: Screenshot comparison (desktop/tablet/mobile)

## 6. Deliverables

- `base.css`: Token declarations (:root)
- 22 CSS files: Progressive conversion to token references
- Responsive media queries: Per major component
- E2E mobile test cases added
- CLAUDE.md: Add undocumented prefixes to CSS prefix rules
