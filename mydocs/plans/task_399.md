# Task 399: 줄바꿈 알고리즘 역공학

## 수행 목표

Task 400에서 해소되지 않은 줄바꿈 불일치를 분석하여 한컴의 줄바꿈 알고리즘과 동일한 결과를 도출한다.

## 배경

Task 400 완료 후 잔여 불일치:
- lseg-04-indent: 줄바꿈 25% (내어쓰기 문단의 text_start ±1~3)
- 기존 샘플: 줄 수 86.7%, 줄바꿈 60.0%
- 기존 샘플 하락 원인: Batang/Gulim 메트릭 추가로 HFT 폰트 충돌

## 구현 계획

### 1단계: lseg-04-indent 불일치 분석

- 내어쓰기 문단에서 text_start ±1~3 차이의 원인 추적
- 내어쓰기 시 available_width 계산 및 줄바꿈 경계 정밀 비교

### 2단계: 기존 샘플 HFT 폰트 메트릭 충돌 해소

- hongbo.hwp 문단 17의 폰트 확인
- Gulim 별칭 매핑이 HFT 폰트(HCR Dotum 등)와 충돌하는지 검증
- 필요 시 별칭 매핑 조건 세분화

### 3단계: 줄바꿈 경계 미세 조정

- 15 HU 허용 오차의 적절성 검증 (더 넓히거나 좁히기)
- 공백 흡수(trailing space) 규칙 정밀화

## 산출물

- 줄바꿈 개선 코드
- 일치율 개선 리포트
