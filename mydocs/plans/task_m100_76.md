### 확장 프로그램의 보안 권한 이점

일반 웹페이지(rhwp-studio 웹 배포)로는 불가능하지만, **확장 프로그램이기 때문에 가능한** 핵심 기능:

| 보안 유형 | 일반 웹의 한계 | 확장 프로그램에서 해결 |
|----------|--------------|---------------------|
| **WASM 실행** | 사이트 CSP가 차단하면 불가 | `wasm-unsafe-eval`로 항상 보장 |
| **CORS 우회** | 서버가 허용해야만 가능 | `host_permissions`로 우회 — 정부24 등 공공사이트 HWP 직접 열기 |
| **로컬 파일 접근** | `file://` 완전 차단 | 사용자 승인 하에 로컬 HWP 파일 직접 열기 |
| **다운로드 제어** | 트리거만 가능 | 저장 경로 지정, HWP 다운로드 감지 → 자동 뷰어 오픈 |
| **클립보드** | 사용자 제스처 필요 | 권한 선언으로 자유로운 텍스트 복사 |
| **웹페이지 HWP 링크 감지** | 불가 | Content Script로 페이지 내 .hwp/.hwpx 링크를 자동 감지, 아이콘 삽입으로 사용자에게 즉시 인지시키고 원클릭으로 열기 |

이 권한들은 원격 서버 기반 웹 서비스로는 제공할 수 없으며, 확장 프로그램이어야만 가능한 사용자 경험이다. 특히 **웹페이지 HWP 링크 감지**는 사용자가 정부24, 학교, 공공기관 사이트를 방문할 때 HWP 파일 링크를 자동으로 인식하여 별도 조작 없이 바로 뷰어로 열어주는 기능으로, 확장 프로그램만이 제공할 수 있는 가장 핵심적인 사용자 경험이다.

### HWP 웹 통합 프로토콜 (`data-hwp-*`)

공공 웹사이트 개발자가 HTML에 `data-hwp-*` 속성을 추가하면 확장 프로그램이 더욱 풍부한 UX를 제공한다. **"한 줄 추가로 시작, 점진적으로 확장"** 원칙.

#### 적용 수준

| 수준 | 개발자 작업 | 확장이 제공하는 UX |
|------|-----------|-------------------|
| **자동 감지** | 없음 (기존 사이트 그대로) | `.hwp`/`.hwpx` 링크 자동 감지, rhwp 아이콘 표시, 원클릭 열기 |
| **최소 적용** | `data-hwp="true"` 1개 추가 | 위 + 명시적 HWP 마킹, 정확한 감지 |
| **표준 적용** | `data-hwp-title/pages/size` 추가 | 위 + 문서 정보 카드, 호버 미리보기, 접근성 향상 |
| **고급 적용** | 썸네일 + 매니페스트 | 위 + 인라인 미리보기, 일괄 다운로드, 기관 브랜딩 |

#### 개별 링크 속성 (`data-hwp-*`)

```html
<!-- 최소: 속성 1개만 추가 -->
<a href="/files/공문.hwp" data-hwp="true">공문.hwp</a>

<!-- 표준: 문서 메타데이터 포함 -->
<a href="/files/공문.hwp"
   data-hwp="true"
   data-hwp-title="2026년 예산 집행 계획"
   data-hwp-pages="12"
   data-hwp-size="245760"
   data-hwp-date="2026-03-15">
  2026년 예산 집행 계획.hwp (240KB, 12쪽)
</a>

<!-- 고급: 미리보기 이미지 + 양식 정보 -->
<a href="/forms/전입신고서.hwp"
   data-hwp="true"
   data-hwp-title="전입신고서"
   data-hwp-pages="2"
   data-hwp-thumbnail="/thumbs/전입신고서.webp"
   data-hwp-form-fields="true"
   data-hwp-category="민원서식">
  전입신고서
</a>
```

#### 페이지 수준 설정 (`<meta>`)

```html
<meta name="hwp:integration" content="enabled">
<meta name="hwp:hover-preview" content="true">
<meta name="hwp:batch-enabled" content="true">
```

#### 점진적 향상 원칙

- `data-hwp-*` 속성은 HTML5 표준으로 브라우저가 무시 → 확장 미설치 시에도 사이트 100% 정상 동작
- 기존 CMS(그누보드, XpressEngine 등) 템플릿에 속성만 추가하면 됨 — 서버 변경 불필요
- 확장 프로그램이 `document.documentElement`에 `data-hwp-extension="rhwp"` 속성을 주입하여 사이트 개발자가 확장 존재를 감지 가능

이 프로토콜의 개발자 가이드를 공식 문서로 제공하여 공공 웹사이트 개발자들이 쉽게 적용할 수 있도록 한다.

### UX 설계 원칙 (시장 조사 + 확장 권한 기반)

1. **제로 허들**: 확장 설치 후 추가 설정 없이 바로 동작
2. **원클릭 열기**: .hwp 파일 클릭/다운로드 → 자동으로 뷰어 탭 오픈
3. **공공사이트 HWP 자동 감지**: 정부24/학교/공공기관 페이지에서 HWP 링크를 자동 인식 → 링크 옆에 rhwp 아이콘 표시 → 즉시 인지 + 원클릭 열기
4. **공공사이트 직접 열기**: CORS 우회로 HWP 첨부파일 바로 열기
5. **로컬 파일 바로 열기**: 브라우저에 HWP 파일을 드래그하면 바로 열림
5. **광고 없음**: 깨끗한 화면
6. **텍스트 복사 가능**: 권한 기반 자유로운 클립보드 접근
7. **대용량 문서 대응**: 페이지 제한 없이, 가상 스크롤로 쾌적하게

## 범위

### 포함

- Manifest V3 확장 프로그램 기본 구조
- 배포 대상: Chrome Web Store + Microsoft Edge Add-ons
- rhwp-studio 전체 탑재 (뷰잉 + 편집)
- HWP + HWPX 파일 열기 (드래그 또는 파일 선택)
- 웹에서 HWP/HWPX 파일 다운로드 시 자동으로 열기
- 웹페이지 내 HWP/HWPX 링크 자동 감지 (Content Script)
  - 페이지 로드 시 DOM 스캔 → .hwp/.hwpx 링크 옆에 rhwp 아이콘/미리보기 버튼 자동 삽입
  - 사용자가 링크를 찾을 필요 없이 시각적으로 즉시 인지
  - 아이콘 클릭 시 뷰어 탭에서 바로 열기
- 저장은 HWP 형식 (HWPX 저장은 추후 구현 예정으로 공지)
- 텍스트 입력/수정, 표 편집, 서식 변경
- 텍스트 선택 + 복사
- 인쇄 기능

- HWP 웹 통합 프로토콜 (`data-hwp-*`) 지원
- 공공 웹사이트 개발자 가이드 문서 제공
