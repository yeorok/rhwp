# Task #68: SVG export 폰트 서브셋 임베딩 — 수행계획서

## 목표

SVG 내보내기 시 사용된 글자의 폰트 서브셋을 추출하여 SVG 내에 `@font-face`로 임베딩한다. 오프라인 환경에서도 한글 텍스트가 올바르게 표시되도록 한다.

## 현상

- 현재 SVG는 inline font-family 속성만 사용, 폰트 데이터 미포함
- 시스템에 해당 폰트가 없으면 한글이 깨져서 표시
- PDF 서브셋 임베딩과 동일한 법적 지위 (출력물)

## 현재 아키텍처

- SVG에 `<style>` 섹션 없음
- `<defs>`는 gradient/clip-path만 포함
- 폰트 이름은 `<text>` 요소의 font-family 속성에 inline
- 폰트 서브셋 crate 미사용

## 구현 단계

### 1단계: 사용된 폰트/글자 수집

- SVG 렌더링 시 `<text>` 요소에서 사용된 (font_family, codepoints) 쌍 수집
- 렌더 트리 순회하여 TextRun 노드에서 font_family + text 추출
- 중복 제거 후 폰트별 codepoint Set 구축

### 2단계: 폰트 서브셋 추출 + base64 임베딩

- 시스템/프로젝트 폰트 파일에서 해당 codepoint만 추출 (서브셋)
- woff2로 변환 후 base64 인코딩
- SVG `<defs>` 앞에 `<style>` 섹션 삽입: `@font-face { font-family: "..."; src: url(data:font/woff2;base64,...); }`
- `--embed-fonts` CLI 옵션으로 on/off

### 3단계: 검증

- 오프라인 환경에서 SVG 열어 한글 텍스트 표시 확인
- `cargo test` 통과
- 기존 SVG 출력 회귀 없음 (기본값은 임베딩 off)

## 의존성

- 폰트 서브셋: `subsetter` crate 또는 `allsorts`
- woff2 변환: `woff2` crate
- 기존: `base64 = "0.22"` (이미 있음)

## 영향 범위

- `src/renderer/svg.rs` — `<style>` 섹션 생성, 폰트 수집
- `src/main.rs` — `--embed-fonts` CLI 옵션
- `Cargo.toml` — 서브셋 crate 추가

## 검증 기준

- `--embed-fonts` 옵션으로 SVG 내보내기 시 `<style>` 내 `@font-face` 포함
- 오프라인 브라우저에서 한글 텍스트 정상 표시
- 기본 동작(임베딩 off)은 기존과 동일
