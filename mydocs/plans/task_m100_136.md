## 문제 요약

`HWP: 디버그 오버레이 보기` 커맨드가 extension host(Node.js)에서 WASM을 직접 로드하여 렌더링함.
Node.js 환경에는 Canvas API가 없어 `measureTextWidth`를 `text.length * 8` 스텁으로 처리하므로,
표 안의 한글 글자 배치가 뷰어와 다르게 틀어짐.

## 원인

| 경로 | measureTextWidth | 배치 정확도 |
|------|-----------------|------------|
| webview (viewer.ts) | Canvas `measureText()` 실제 호출 | 정확 |
| extension host (extension.ts) | `text.length * 8` 스텁 | 부정확 |

## 개선 방향

`cmdDebugOverlay`를 extension host 직접 렌더링 방식에서 **webview 위임 방식**으로 전환.

### 메시지 흐름

```
extension host                    webview (viewer.ts)
     │                                   │
     │── { type: "exportDebugOverlay" } ──▶│
     │                                   │ set_debug_overlay(true)
     │                                   │ renderPageSvg() × N
     │                                   │ set_debug_overlay(false)
     │◀── { type: "debugOverlaySvgs",   ──│
     │       svgs: string[] }             │
     │                                   │
     │ HTML 조합 → /tmp 저장 → vscode.open
```

### 파일 미오픈 시 처리

현재 `sendPrint`와 동일한 패턴 적용:
- 뷰어가 열려있으면 → 즉시 메시지 전송
- 뷰어 미오픈 시 → `vscode.openWith`로 열고, `ready` 수신 후 `pendingDebugOverlay`에서 처리

## 구현 범위

- `rhwp-vscode/src/extension.ts` — `cmdDebugOverlay` 수정: webview에 메시지 전송 + SVG 수신 대기
- `rhwp-vscode/src/hwp-editor-provider.ts` — `sendDebugOverlay()` 메서드 추가, `pendingDebugOverlay` Set 추가, SVG 응답 콜백 처리
- `rhwp-vscode/src/webview/viewer.ts` — `exportDebugOverlay` 메시지 처리 추가

## 구현 단계

| 단계 | 내용 |
|------|------|
| 1 | webview: `exportDebugOverlay` 메시지 수신 → SVG 렌더링 → `debugOverlaySvgs` 응답 |
| 2 | hwp-editor-provider: `sendDebugOverlay()` + `pendingDebugOverlay` + SVG 수신 콜백 |
| 3 | extension.ts: `cmdDebugOverlay` webview 위임 방식으로 전환, HTML 생성/저장은 유지 |

## 기대 효과

- 디버그 오버레이 글자 배치가 뷰어와 동일하게 정확해짐
- extension host의 `measureTextWidth` 스텁 의존 제거 (디버그 오버레이 한정)
