# Task #70: VSCode 확장 webview 폰트 로딩 현행화 — 수행계획서

## 목표

VSCode 확장의 webview에 font-loader/font-substitution을 연동하여 HWP 문서의 한글 폰트가 올바르게 렌더링되도록 한다.

## 현상

- VSCode webview(viewer.ts)에 폰트 로딩 로직 없음
- 시스템 폰트에만 의존 → 시스템에 해당 폰트 없으면 한글 깨짐
- rhwp-studio의 font-loader.ts 변경 사항(#67) 미반영

## 구현 단계

### 1단계: 오픈소스 폰트를 확장 번들에 포함

- webpack CopyPlugin으로 `web/fonts/` 오픈소스 woff2를 `dist/media/fonts/`에 복사
- 함초롬체 CDN은 CSP에 jsdelivr 도메인 추가

### 2단계: webview HTML에 @font-face 등록

- `getHtml()`에 `<style>` 내 @font-face 규칙 생성
- 로컬 폰트: `webview.asWebviewUri()`로 확장 내 woff2 경로 생성
- CDN 폰트: jsdelivr URL 직접 참조
- CSP `font-src`에 CDN 도메인 추가

### 3단계: 검증

- VSCode에서 HWP 문서 열어 한글 렌더링 확인
- 시스템에 한글 폰트 없는 환경에서 오픈소스 폰트 폴백 확인

## 영향 범위

- `rhwp-vscode/webpack.config.js` — 폰트 파일 복사
- `rhwp-vscode/src/hwp-editor-provider.ts` — CSP, @font-face 추가
- `rhwp-vscode/.vscodeignore` — 폰트 파일 포함 확인
